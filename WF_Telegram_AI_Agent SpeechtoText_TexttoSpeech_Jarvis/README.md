# WF_Telegram_AI_Agent (Speech-to-Text & Text-to-Speech - Jarvis)

## Overview
This n8n workflow is a **Telegram-based AI assistant** that handles both text and voice interactions. It can:
- ✅ Process text messages with AI responses
- 🎤 Convert speech (voice notes) to text
- 🔊 Generate audio responses from text
- 📡 Search the web using SerpAPI
- 📧 Send emails via Gmail
- 💾 Maintain conversation memory per user

---

## Workflow Architecture

### Two Main Flows:

#### **Flow 1: Text-to-Text Reply**
```
Telegram Trigger → Switch (Text Detected) → Edit Fields → AI Agent → Send Text Message
```

#### **Flow 2: Speech-to-Speech Reply (STT → TTS)**
```
Telegram Trigger → Switch (Voice Detected) → Get File → Transcribe → Optimize Message → Generate Audio → Send Audio
```

3. **SerpAPI Key** (Optional - for web search)

4. **Gmail OAuth2** (Optional - for email sending)
   - Enable Gmail API in Google Cloud Console
   - Set up OAuth2 credentials
   - Store as `Gmail account` credential

## Node-by-Node Breakdown

### **1. Telegram Trigger** 🔔
- **Type:** Webhook Listener
- **Function:** Listens for incoming Telegram messages (text & voice)
- **Triggers on:** `message` updates
- **Webhook ID:** `5e0f7fdc-02e1-4649-950e-e6fb06dbb755`
- **Credential:** `se_tgllmbot`

---

### **2. Switch Node** 🔀
- **Type:** Conditional Router
- **Function:** Routes based on message type
- **Conditions:**
  - **Output 1 (Text):** If `$json.message.text` exists → sends to text flow
  - **Output 2 (Speech):** If `$json.message.voice.file_id` exists → sends to speech flow

---

## Text-to-Text Flow

### **3. Edit Fields**
- **Type:** Set/Edit node
- **Function:** Extracts and formats the text message
- **Sets:** `message.text` = user's input message
- **Output:** Cleaned message data to AI Agent

---

### **4. AI Agent** 🤖
- **Type:** LangChain Agent
- **Model:** GPT-4o-mini
- **Prompt Type:** Define
- **System Instructions:**
  ```
  "You are a helpful personal assistant. Able to call tools based on user requirements:
  - SerpAPI: Call when user requests latest news for web search
  - Gmail: Call when user wants to send emails
  - Email format: Professional, notify user of send status (success/failure)"
  ```
- **Connected Tools:** SerpAPI, Gmail
- **Memory:** Simple Memory (per chat/user)
- **Output:** Text response to send back

---

### **5. OpenAI Chat Model** 💬
- **Type:** LangChain Chat Model
- **Model:** `gpt-4o-mini`
- **Function:** Powers the AI Agent with language capabilities
- **Credential:** `mohanai` (OpenAI API)

---

### **6. Simple Memory** 🧠
- **Type:** Buffer Window Memory
- **Session ID Type:** Custom Key
- **Session Key:** User's Telegram chat ID (`$('Telegram Trigger').item.json.message.chat.id`)
- **Function:** Maintains conversation history per user
- **Behavior:** Remembers context across multiple messages

---

### **7. SerpAPI Tool** 🔍
- **Type:** LangChain Tool
- **Function:** Web search capability
- **When Used:** AI Agent calls this when user asks for latest news
- **Credential:** `SerpAPI account`

---

### **8. Gmail Tool** ✉️
- **Type:** Gmail Node
- **Function:** Send emails on demand
- **Parameters:**
  - `sendTo`: Extracted by AI
  - `subject`: Generated by AI
  - `message`: Created by AI
- **Credential:** `Gmail account`
- **Note:** AI uses `$fromAI()` to extract email details from user prompt

---

### **9. Send a Text Message** 📤
- **Type:** Telegram Send
- **Function:** Sends AI response back to user as text
- **Parameters:**
  - `chatId`: User's Telegram chat ID
  - `text`: AI Agent's output response
- **Credential:** `se_tgllmbot`

---

## Speech-to-Speech Flow

### **10. Get a File** 📥
- **Type:** Telegram File Download
- **Function:** Retrieves the voice note file from Telegram servers
- **Input:** Voice file ID from user's message
- **Output:** File data ready for transcription
- **Credential:** `se_tgllmbot`

---

### **11. Transcribe a Recording** 🎙️ → 📝
- **Type:** OpenAI Whisper API
- **Function:** Converts voice/audio to text
- **Operation:** `transcribe`
- **Input:** Audio file from Telegram
- **Output:** Transcribed text
- **Credential:** `mohanai` (OpenAI API)

---

### **12. Message a Model** 📊
- **Type:** OpenAI Chat
- **Function:** Optimizes transcribed text for audio response
- **System Message:**
  ```
  "Summarize the input with high accuracy ensuring high clarity.
  Focus on delivering contextual and optimize for speech synthesis.
  Remove redundant phrases, introductory statements and closing remarks.
  The OUTPUT should be natural and engaging for an audio format."
  ```
- **Input:** Transcribed text
- **Output:** Optimized, concise message ready for voice synthesis
- **Model:** `gpt-4o-mini`

---

### **13. Generate Audio** 🔊
- **Type:** OpenAI Text-to-Speech
- **Function:** Converts optimized text to MP3 audio
- **Input:** Output from "Message a model" node
- **Output:** Audio file (MP3 format)
- **Credential:** `mohanai` (OpenAI API)

---

### **14. Send an Audio File** 📮
- **Type:** Telegram Audio Send
- **Function:** Sends generated audio back to user
- **Parameters:**
  - `chatId`: User's Telegram chat ID
  - `binaryData`: `true` (sends audio as file)
- **Input:** Generated audio from previous node
- **Credential:** `se_tgllmbot`

## How to Use This Workflow

### **For Text Messages:**
1. User sends text to bot → Trigger catches it
2. Switch detects text → Routes to "Edit Fields"
3. "Edit Fields" cleans the message
4. "AI Agent" processes with context from "Simple Memory"
5. Response sent back as text message
6. Memory updated for future context

### **For Voice Messages:**
1. User sends voice note to bot → Trigger catches it
2. Switch detects voice → Routes to "Get a File"
3. "Get a File" downloads the audio
4. "Transcribe" converts audio to text
5. "Message a Model" optimizes the text for speech
6. "Generate Audio" creates MP3 response
7. "Send an Audio File" delivers the reply
8. User receives natural-sounding audio response

### **For Special Requests:**
- **Latest News:** User asks "What's the news?" → AI calls **SerpAPI**
- **Send Email:** User asks "Send an email to..." → AI calls **Gmail Tool**

## Troubleshooting

### Issue: Bot doesn't respond to messages
- ✅ Check Telegram credential is valid
- ✅ Ensure webhook is active (green checkmark)
- ✅ Verify OpenAI API key is working

### Issue: Voice transcription fails
- ✅ Check OpenAI quota/balance
- ✅ Ensure audio file is in supported format
- ✅ Verify "Get a File" downloads properly

### Issue: AI Agent not calling tools
- ✅ Check SerpAPI/Gmail credentials are set up
- ✅ Verify tool descriptions are clear in system prompt
- ✅ Test with explicit tool requests (e.g., "Search for...")

### Issue: Memory not persisting
- ✅ Verify session key correctly extracts chat ID
- ✅ Check Simple Memory node is connected to AI Agent
- ✅ Clear memory if needed (delete session in n8n)
